# 🚀 QA Automation Journey: Mi Portafolio de Aprendizaje

¡Bienvenido! Este repositorio es el espacio donde documento mi evolución como **QA Automation Engineer**. Aquí no solo guardo código, sino que registro mi proceso de aprendizaje autodidacta, construyendo soluciones de pruebas automatizadas desde las bases.

---

## 🎯 Mi Misión
Mi objetivo es dominar las herramientas líderes del mercado para asegurar la calidad del software. Este proyecto nació de la curiosidad y las ganas de entender cómo "romper" y "validar" aplicaciones de forma eficiente.

## 📂 Estructura del Proyecto
He organizado este repositorio por tecnologías y especialidades para mantener un aprendizaje estructurado:

### 🤖 Robot Framework
*   **[API](./robotframework/api):** Automatización de servicios REST usando `RequestsLibrary`.
*   **[Web](./robotframework/web):** Pruebas de interfaz de usuario con `SeleniumLibrary`.
*   **[Mobile](./robotframework/mobile):** (En construcción) Pruebas en dispositivos móviles con `Appium`.

### 🎭 Próximamente (Roadmap)
*   **Playwright:** Explorando el futuro de la automatización Web y API con TypeScript/Python.
*   **Cypress:** Pruebas E2E modernas.

---

## 🛠️ Tecnologías y Herramientas
*   **Lenguaje:** Python 3.x
*   **Framework:** Robot Framework
*   **CI/CD:** GitHub Actions (Ejecución automática de pruebas)
*   **Editor:** VS Code

## 📝 Filosofía de Código
Cada línea de código en este repositorio incluye **comentarios detallados**. ¿Por qué?
1.  Para reforzar mi propio aprendizaje (explicarlo es entenderlo).
2.  Para que cualquier persona que visite mi perfil pueda seguir la lógica sin perderse.

Este proyecto cuenta con **GitHub Actions**. Puedes ir a la pestaña **[Actions]** de este repositorio para ver cómo se ejecutan los tests automáticamente con cada cambio que realizo.
